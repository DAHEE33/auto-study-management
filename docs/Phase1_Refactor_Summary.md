# 🔄 Study-Sync Phase 1 리팩토링 서머리 (2026-04-15)

본 문서는 새로운 `설계.md` 기획에 맞추어 카카오 챗봇 연동 및 구글 시트 코어 로직(Step 1 ~ Step 3)을 성공적으로 리팩토링한 결과와, 다음 작업 시 진행할 테스트 방법을 안내합니다.

## 🎯 1. 주요 변경 사항 및 확인해야 할 핵심 파일

### 1) DB 스키마 갱신 및 UserKey 식별 (`integrations/google_sheets.py`)
- 구글 스프레드시트 초기화 로직(`setup_initial_data`)이 **최신 설계 스키마**에 맞게 업데이트 되었습니다. (예: `Member_Master`에 남은월휴, `Daily_Log`에 승인여부 컬럼 추가)
- 닉네임 대신 카카오톡이 전송하는 고유 `UserKey`를 통해 멤버를 검색하고 상태를 수정하는 `get_member_by_userkey()` 함수가 새롭게 도입되었습니다.

### 2) 비즈니스(규칙) 엔진 분리 (`services/check_in_engine.py`)
- **이전:** 모든 로직이 `settlement_engine.py` (정산 엔진)에 뭉쳐 있었습니다.
- **개선:** 인증 유효성을 검사하는 `check_in_engine.py`를 신설했습니다.
  - `is_within_deadline()`: 새벽 5시를 기준으로 당일/익일 귀속을 나누고 새벽 1시 지각을 판독합니다.
  - `process_leave_request()`: 잔여 휴무(주휴, 반휴, 월휴)가 부족하면 카톡 답변으로 컷(차단)하고, 충분하면 시트에서 값을 깎도록 허가하는 로직이 완성되었습니다.

### 3) 챗봇 라우터 정리 (`routers/webhook.py`)
- 카카오톡 챗봇에서 들어오는 블록명/발화명을 바탕으로 **각 인증 과정이 완벽히 분기(If-Else)** 되도록 최적화했습니다.
- 예: 
  - "주휴" 클릭 👉 사진 없이 즉시 엔진을 통해 휴무 감차 후 "푹 쉬세요" 문자 발송.
  - "특휴" 클릭 👉 사진(증빙)을 드라이브에 저장한 뒤, 시트에 '승인대기(대기)' 로 기록.

## 🧪 2. 내일 바로 진행해야 할 테스트 (How to Test)

1️⃣ **가장 먼저: 구글 시트 비우기 확인**
   - 연결된 구글 스프레드시트에 접속하셔서 예존에 테스트하던 탭(`Member_Master`, `Daily_Log` 등)이 모두 백지상태인지 확인해주세요. (빈 시트여야 최신화 세팅이 정상 발동합니다.)

2️⃣ **서버 구동 및 뼈대 주입 확인**
   - 터미널을 열고 서버를 실행하세요.
     ```bash
     uvicorn main:app --reload --port 8000
     ```
   - 서버가 켜질 때 터미널에 _"✔️ 'Member_Master' 시트에 기초 데이터 삽입 완료"_ 라는 메시지가 뜨고, 구글 시트에 예쁜 최신 양식 표가 짜잔! 하고 생성되는지 확인합니다.

3️⃣ **카카오톡 챗봇 테스트 (개인 폰으로 접속)**
   - `ngrok`을 켜서 외부 주소를 얻은 뒤 카카오 오픈빌더 스킬 서버 URL에 적용해주세요. (`ngrok http 8000`)
   - 카톡에서 봇에게 말을 걸거나 하단 퀵 플러그인 버튼들을 눌러봅니다.
     - `[내 현황 확인]` 클릭 👉 닉네임과 휴무 횟수, 예치금이 즉시 봇 답장으로 오는지 핑퐁 테스트.
     - `[주휴 사용]` 클릭 👉 챗봇이 처리 완료됐다고 안내하고, '구글 시트'의 `Member_Master`에서 내 주간휴무 숫자가 0으로 자동 변경되는지 실시간 확인.
     - *방어막 테스트:* 두 번 연속 `[주휴 사용]` 클릭 👉 챗봇 엔진이 "이번 주 휴무를 모두 소진하셨습니다" 라고 방어막을 제대로 치는지 확인.

# 🧪 TESTING.md - 통합 점검 및 "진짜" 실제 운영 테스트 가이드

이 문서는 모의(Mock) 테스트를 넘어서, **실제 카카오톡 서버**와 로컬 파이썬 서버를 바로 연동하여 운영 수준의 **End-to-End (E2E) 테스트**를 진행하는 방법을 안내합니다.

---

## [Step 1 테스트] 공휴일/점검 예외 상황 및 결석자 색출
> 구글 시트의 `Admin_Calendar` 스위치가 올바르게 작동하여 자율참여일에는 결석 벌금을 매기지 않는지 테스트합니다.

1. `Admin_Calendar` 시트의 어제 날짜에 "자율참여" 이벤트를 적어둡니다.
2. 터미널에서 다음 스크립트를 수동으로 실행합니다.
   ```bash
   python jobs/daily_absence.py
   ```
3. **[검증 포인트]**: 터미널에 "자율참여로 지정되었습니다. 결석 벌금을 매기지 않습니다" 유사 메시지가 나타나며, `Daily_Log` 시트에 벌금 내역(-2000원)이 강제로 생성되지 않으면 성공입니다!

---

## [Step 2 테스트] 카카오톡 인증 분기 처리 (반휴/주휴/일반)
> 로컬 서버의 기능을 카카오톡 챗봇과 직결하여, **단번에 사진과 텍스트를 전송했을 때** 시스템이 어떻게 알아듣고 반영하는지 검증합니다.

### 2.1 서버 가동 및 연동 준비
1. 터미널 1을 열고 웹 서버를 켭니다: `uvicorn main:app --reload --port 8000`
2. 터미널 2를 열고 외부 핑뚫기: `ngrok http 8000`
3. 오픈빌더 스킬 설정: ngrok 화면에 뜬 `https://xxxx.ngrok-free.app/api/v1/webhook` 주소를 스킬에 연동.

### 2.2 인증 시나리오 검증 리스트
1. **[반휴/일반 텍스트+사진 동시 전송 테스트] (핑퐁 없이 사진과 키워드 한 번에!)**
   - 폰에서 봇 채팅방에 **사진 한 장과 "반휴 사용"이라는 텍스트를 한 번에** 묶어서 전송합니다.
   - **[검증 포인트]**: 봇이 되묻기 없이 즉시 알아듣고 "반휴가 적용되어 목표시간 1h 컷으로 채점되었습니다" 응답 후, 시트에 주간휴무가 -0.5명 깎여있어야 성공입니다.
   - 텍스트 없이 사진만 전송하면 자동으로 **[일반 인증]** 기준으로 2h 짜리 채점이 돌아가는지 확인합니다.

2. **[주휴/월휴 버튼 테스트]**
   - 봇 하단 메뉴에서 `[🏖️ 주휴/월휴 사용]` 퀵 리플라이 클릭.
   - **[검증 포인트]**: 사진을 달라는 요구 없이 즉시 "휴무 처리가 완료되었습니다!" 메시지가 오며, `Daily_Log` 시트에 사진이 빈칸인 상태로 PASS 처리되어야 합니다.

3. **[내 현황 조회 검증]**
   - `[📈 내 현황 확인]` 퀵 리플라이 클릭.
   - **[검증 포인트]**: 봇이 시트를 긁어와서 **(주별 누적 공부시간, 주별 낸 벌금, 주간/월간 남은 휴무, 남은 예치금, 월별 누적 공부시간)** 이 5가지 지표를 예쁘게 말풍선으로 응답하는지 확인합니다.

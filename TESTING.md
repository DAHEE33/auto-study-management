# 🧪 TESTING.md - 통합 점검 및 "진짜" 실제 운영 테스트 가이드

이 문서는 모의(Mock) 테스트를 넘어서, **실제 카카오톡 서버**와 로컬 파이썬 서버를 바로 연동하여 운영 수준의 **End-to-End (E2E) 테스트**를 진행하는 방법을 안내합니다.

---

## 1. 🌐 카카오 i 오픈빌더 + Ngrok 실제 연동 테스트 (핵심)
> 로컬 환경(내 컴퓨터)에서 켜진 파이썬 서버를 외부의 카카오톡 서버가 접속할 수 있도록 구멍을 뚫어(ngrok), 핸드폰 카카오톡으로 사진을 전송하고 구글 시트에 등재되는 전체 과정을 테스트합니다.

### Step A: 서버와 터널 가동하기
1. 터미널 1을 열고 웹 서버를 켭니다.
   ```bash
   uvicorn main:app --reload --port 8000
   ```
2. 터미널 2를 새로 열고 **Ngrok**을 이용해 외부 접속용 주소를 발급받습니다.
   ```bash
   ngrok http 8000
   ```
   * 실행 후 화면에 나타나는 `https://xxxx-xxxx.ngrok-free.app` 형태의 주소를 복사합니다.

### Step B: 카카오톡 오픈빌더에 스킬 연결하기
1. 카카오 i 오픈빌더 어드민 페이지에 접속합니다.
2. 좌측 메뉴에서 **스킬(Skill)** 탭으로 들어가 새로운 스킬을 생성합니다.
3. 스킬 URL 창에 **복사한 ngrok 주소 + `/api/v1/webhook`** 을 붙여넣습니다.
   * 예) `https://xxxx-xxxx.ngrok-free.app/api/v1/webhook`
4. 봇의 '오늘 인증' 블록(시나리오)에서 파라미터 설정을 켭니다.
   * 이미지를 받을 때 `sys.image` 플러그인을 활용하여 이미지를 파라미터로 떨어뜨리도록 설정한 뒤, 위의 스킬을 호출하게 연결합니다.

### Step C: 내 폰에서 봇에게 사진 보내보기 (최종 검증)
1. 카카오톡을 켜고 채널 봇에게 인증용 사진(구루미 타이머가 찍힌 스크린샷)을 전송합니다.
2. 봇이 스마트 응답 메뉴를 주면 `[🔥 오늘 인증]` 을 누릅니다.
3. **[성공 기대 효과]**:
   - 봇이 "이미지를 수신했습니다..." 라고 응답한 뒤 약 1~2초 후.
   - **"✅ 인증 완료! (판정: 통과, 종료시각: 23:55 ...)"** 이라는 최종 응답 말풍선을 돌려줍니다!
   - 즉시 구글 시트의 `Daily_Log` 시트를 열어보면 방금 올린 사진의 구글 드라이브 링크와 함께 벌금 내역이 완벽하게 전산 등록된 것을 확인할 수 있습니다.

## 2. ⚖️ [보조 배치] 매일 정오 결석자 자동 색출
> 위의 카카오 연동 테스트가 끝난 후, 인증하지 않은 사람들에게 벌금을 먹이는 기능 테스트.

**실행 명령어:**
```bash
python jobs/daily_absence.py
```
- **테스트 방법**: `Member_Master` 구글 시트에 "lazy_user" 같은 빈 유저명 한 줄을 추가하고 위 명령어를 돌리면, 카카오에 오늘 인증을 안 한 사람들의 벌금(-2000원) 내역이 시트에 바로 찍힙니다.

---

## 3. 🛡️ [보조 배치] 구글 시트 관리자 승인 (수동 조작 동기화)
> 멤버가 예비군/시험 등 특수 휴가를 올려서 구글 시트에 Pending 상태로 기록된 경우 방장이 처리.

**실행 명령어:**
```bash
python jobs/admin_sync.py
```
- **테스트 방법**: 관리자가 직접 구글 시트에 접속해 `Pending` 칸을 `승인(TRUE)`으로 수정하고 위 스크립트를 누르면 봇이 유저에게 카톡을 보낼 채비를 합니다.

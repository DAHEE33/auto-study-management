# 🧪 TESTING.md - 통합 점검 가이드 및 시나리오 테스트 명세서

이 문서는 개발이 완료된 `Study-Sync` 자동화 스터디 봇을 **어떤 상황에서, 언제, 어떤 스크립트로** 테스트하고 검증(모의 실행)해볼 수 있는지 상세한 시나리오별 안내를 제공합니다.

---

## 1. 🌐 카카오톡 응답 봇 테스트 (대시보드 포함)
> 유저의 카카오톡 메시지에 봇이 응답을 반환할 수 있는지, 대시보드 화면이 잘 작동하는지 확인하는 테스트입니다.

**어떤 파이썬 스크립트?**
`main.py` (FastAPI 서버 구동)

**실행 명령어:**
```bash
uvicorn main:app --reload --port 8000
```
- **실행 상황**: 프론트엔드 연동 전 서버가 살아있는지 검사할 때, 혹은 관리자가 통계 대시보드를 직접 눈으로 보고 싶을 때 실행합니다.
- **테스트 방법**: 브라우저에서 `http://127.0.0.1:8000/dashboard` 로 접속하여 구글 시트의 벌금 데이터와 UI가 실시간 연동되어 렌더링되는지 눈으로 확인합니다. 카카오톡 API 동작을 확인하려면 API 테스트 툴(Postman)로 `POST http://127.0.0.1:8000/api/v1/webhook` 주소에 JSON 바디를 날려봅니다.

---

## 2. ⚖️ [배치 - 스케줄러] 매일 정오 결석자 자동 색출 테스트
> 어제 시간대에 공부 인증 사진을 업로드하지 않아 로그(Daily_Log)가 없는 회원을 찾아내 구글 시트에 -2000원 벌금을 때리는 기능 테스트.

**어떤 파이썬 스크립트?**
`jobs/daily_absence.py`

**실행 명령어:**
```bash
python jobs/daily_absence.py
```
- **실행 상황**: 밤사이 아무도 눈치채지 못하도록 무단 결석한 사람을 스케줄러가 잘 색출해서 벌금을 먹이는지 확인하고 싶을 때.
- **테스트 방법**:
  1. 먼저 `Member_Master` 구글 시트에 "lazy_user" 같은 빈 유저명 한 줄을 추가합니다.
  2. 위 명령어를 실행합니다.
  3. `Daily_Log` 구글 시트 맨 아래에 `lazy_user`가 "결석 / -2000원" 이라는 낙인 행이 추가되었는지 육안으로 확인합니다.

## 3. 🛡️ [배치 - 모니터링] 구글 시트 관리자 승인 동기화 테스트
> 멤버가 예비군/시험 등 특수 휴가를 올려서 구글 시트에 Pending 상태로 기록된 경우, 방장이 이를 '승인' 버튼 클릭 시 카카오톡으로 알려주는 기능.

**어떤 파이썬 스크립트?**
`jobs/admin_sync.py`

**실행 명령어:**
```bash
python jobs/admin_sync.py
```
- **실행 상황**: 관리자가 구글 시트에 접속해서 수동으로 무언가를 끄적였을 때, 시스템이 이를 재빠르게 훑어보고 유저들에게 메시지를 날리는 이벤트 동기화 로직을 볼 때 거칩니다.
- **테스트 방법**:
  1. `Daily_Log` 시트에 임의로 (판정=Pending, 승인=TRUE) 인 행을 한 줄 생성합니다.
  2. 명령어 실행.
  3. 터미널 창에 `[감지] 관리자가 ~~~님의 사유를 방금 승인했습니다!`라는 텍스트 트래킹이 발생하는지 점검합니다.

## 4. 🧠 코어 정산 로직 및 OCR 엔진 단위 테스트
> 위 1, 2, 3이 동작하기 위한 "두뇌" 파트가 정상 작동하는지 점검. 1시 이전 종료 룰, 지각벌금, OCR 예외 처리가 작동하는지 점검.

**어떤 파이썬 스크립트?**
`test_step2.py`

**실행 명령어:**
```bash
python test_step2.py
```
- **실행 상황**: 벌금 산정 룰이 변경되거나, 익일 낮 12시인지 오후 2시 마감인지에 대한 '시간 규칙'을 변경했을 때, 이것이 톱니바퀴처럼 정상 계산하는지 점검할 때 쓰입니다.
- **테스트 방법**: 출력되는 시나리오 모의 결과를 읽습니다. (`1h 30m -> Expected: -500`)

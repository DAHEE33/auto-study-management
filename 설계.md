# 🚀 Project: Study-Sync (Auto-Settlement Engine)

> **"관리 리소스 Zero를 지향하는 스터디 운영 자동화 솔루션"**
> 본 프로젝트는 카카오톡 챗봇, OCR, Google Sheets API를 결합하여 출석 인증, 벌금 정산, 스토리지 관리를 자동화하는 **Serverless 정산 시스템**입니다.

---

## 1. 🏗 System Architecture
* **Backend**: Python 3.11 / FastAPI (Asynchronous)
* **Database**: Google Sheets API (Source of Truth / NoSQL Interface)
* **Storage**: Google Drive API (Image Hosting & Lifecycle)
* **OCR**: Google Cloud Vision API (Parsing Gurumi UI)
* **Infrastructure**: Railway (PaaS), GitHub Actions (Cron Jobs)

---

## 2. 📅 핵심 운영 및 시간 검증 로직

### 2.1 이중 시간 검증 (Dual-Time Validation)
* **제출 마감**: 다음날 정오 12:00 까지 (이후 제출 시 결석 처리, 이후 유형 변경 불가)
* **공부 종료/달성 시간**: 당일 01:00 이전 종료 필수 (1시까지 목표시간 완료 여부 OCR 확인)
* **출석 대체 사진**: 전날 마감 시간 또는 구루미 내부의 '출석 사진'만 인정 (그외 불가)

### 2.2 휴무 및 인증 유형 (Tranactions)
* **[🔥 일반인증 (공부)]**:
  * 구루미 내 본인 화면 타이머 캡처 사진 제출. 최소 2h 또는 닉네임별 설정 목록(고정 목표시간) 달성 필수.
  * 목표 시간 도달 시 정상 통과. 미달 시 벌금 부과.
* **[🌗 반휴사용]**:
  * 주 1회 주휴를 반휴 2회로 분할 가능.
  * 사용 시 당일 목표시간은 **1h**로 고정됨 (원래 목표가 3h인 유저도 1h만 달성하면 됨).
* **[🏖️ 주휴/월휴]**:
  * **주휴**: 주 1회 (반휴 미사용 시)
  * **월휴**: 월 1회 (관리자 승인 하에 특휴로 대체 변경 가능)
  * 사용 시 인증 사진 없이 즉시 출석 인정(PASS).
* **[🏥 특휴(증빙 필요)]**:
  * 시험(당일 휴무 인정), 병가(월 1회, 처방전 필요), 2달 이내 휴가제 등.
  * 챗봇을 통해 증빙 사진 업로드 → AI는 판독하지 않고 **관리자 승인 대기(Pending)** 처리 → 관리자가 시트 확인 후 최종 승인.

## 3. 💰 벌금 및 정산 로직 (Zero-Sum Engine)

### 3.1 벌금 산정 기준 (Penalty)
* **결석**: -2,000원 (미인증, 익일 12시 권장 시간 초과)
* **시간 미달 (일반 인증 시)**:
  * 인증 시간 1시간 이상, 최소 목표시간 미달: -500원
  * 인증 시간 1시간 미만 (심각 미달): -1,000원
* **허위/거짓 인증**: 
  * 날짜를 잘못 올린 경우: -1,000원
  
### 3.2 주간 결산 및 알림 (토요일 정오)
* **방식**: 매주 토요일 정오(12:00)에 한 주간의 벌금 통계를 정산하여 시트에 완료 기록을 남기고, 카카오톡 알림(알림톡 또는 푸시)을 통해 '한 주 마무리 기록'을 멤버들에게 전송합니다. 이후 벌금을 내지 않은 사람을 기준으로 누적된 벌금을 1/n로 나누어 배분하고 예치금에 반영합니다.
* **상금**: 상금(잔돈 등) 액수가 3,000원 이상일 시 관리자 재량으로 송금.
* **휴가 관련 위약**: 가입 첫 4주 내 (첫 달) 휴가 사용 불가. 부득이하게 사용할 경우 휴가 이후 예치금 반환일자 리셋 (잦은 휴가 시 강퇴 조치 고려).
* **미반환 예치금**: 두 달 이상 참여자에게 1/n 배분.

## 4. 🎛 공휴일 및 서버 점검 등 예외 상황 (관리자 스위치 제어)

시스템의 유연한 대처를 위해 관리자가 지정한 예외 상황은 복잡한 조건 없이 모두 **전체 자율 참여**로 일괄 통일하여 적용됩니다.

* **공휴일(빨간날) 및 다수 오류/서버 점검 (시간 무관)**:
  * 관리자가 시트에 기입한 날짜는 점검 시간에 관계없이 **전체 자율 참여**로 적용됩니다.
  * 다음날 정오에 인증하지 않은 인원이 있어도 결석 처리나 벌금(-2000원)을 부여하지 않습니다.
* **자율 참여를 알리는 UI**: 적용 당일 사용자가 구루미 봇 접속 시, 시트의 스위치를 인식하여 *"오늘은 점검/공휴일로 인해 자율참여 입니다"* 라고 자동으로 안내 멘트가 노출됩니다.
* **개별적 오류 발생**: 소수에게 나타나는 오류는 톡봇으로 해결 불가하므로 방장 갠톡 후, 구글 시트에서 해당 유저의 벌금액만을 관리자가 수동으로 0원 처리합니다.

## 5. 📱 User Interface (카카오톡 챗봇 UX)

사용자가 자신의 상태에 맞게 수월히 조치할 수 있도록 챗봇 하단 메뉴(Quick Reply)를 통한 **유형 선택 트리**를 제공합니다.

### 5.1 스마트 메뉴 Flow
* `[오늘의 인증]`
  ↳ `[🔥 일반 공부 인증]` → ("사진 제출바랍니다" → OCR 채점)
  ↳ `[🌗 반휴 사용]` → ("반휴 차감. 오늘 목표 1h. 사진 제출바랍니다" → 1h 컷 기준 OCR 채점)
  ↳ `[🏖️ 주휴/월휴 사용]` → ("휴무 차감. 완료되었습니다!" → 사진 없이 즉시 종료)
  ↳ `[🏥 특휴(병가/시험) 증빙제출]` → ("처방전/수험표 제출바랍니다" → 사진만 받아 관리자 대기 상태로 이관)
* `[📈 내 현황 확인]` (개인 누적 통계 조회)
  ↳ 다음 정보를 즉시 응답: **주별 누적 공부시간, 주별 누적 낸 벌금, 주간/월간 남은 휴무 횟수, 남은 예치금(원금-벌금), 월별 누적 공부시간**.

## 6. 🛠 Admin Control (운영 관리 로직)

관리자(방장)는 Google Sheets 를 데이터베이스이자 스위치 보드로 활용합니다.

* **특휴 승인**: `Daily_Log`에서 상태가 `Pending`인 건을 확인하고 증빙 사진(URL)을 본 후 체크박스 승인 클릭.
* **시트 기반 강제 조정**: 개별 에러, 정산 오류 발생 시 관리자가 `Daily_Log` 시트 벌금 내역만 직접 수정하면 차주 정산에 즉시 반영.

## 7. 🗄 Database Schema (Google Sheets)

### Sheet 1: `Member_Master`
| 닉네임 | UserKey | 상태 | 목표시간(m) | 주간휴무_잔여량 | 남은월휴 | 예치금 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| dev_user | UK123 | 활동 | 120 | 1.0 | 1 | 10,000 |

*(💡 주간휴무_잔여량은 매주 월요일 00시에 `1.0`으로 자동 리셋됩니다. 반휴 사용 시 `0.5` 차감, 주휴 사용 시 `1.0`이 차감되어 하나의 지표로 헷갈림 없이 관리됩니다.)*

### Sheet 2: `Daily_Log` (트랜잭션)
| 날짜 | 닉네임 | 인증유형(일반/반휴/주휴/특휴) | 판정결과 | 승인여부(특휴시) | 벌금액 | 제출시각 | 사진URL |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 2026-04-06 | 홍길동 | 일반 | 통과 | - | 0 | 09:20 | [Link] |
| 2026-04-07 | 김철수 | 특휴 | 대기 | N | 0 | 11:10 | [Link] |

### Sheet 3: `Admin_Calendar` (스위치 설정 시트)
관리자가 미리 예외 상황 날짜를 기입해 두면 봇이 이 시트를 참조하여 안내 멘트와 판정 기준을 자동 조율합니다.

| 날짜 | 이벤트명 | 룰 변경 타입 (적용방식) |
| :--- | :--- | :--- |
| 2026-05-05 | 어린이날 | 자율참여 (미인증시 벌금 면제) |
| 2026-05-15 | 구루미 서버점검 | 자율참여 (미인증시 벌금 면제) |

# 🚀 Project: Study-Sync (Auto-Settlement Engine)

> **"관리 리소스 Zero를 지향하는 스터디 운영 자동화 솔루션"**
> 본 프로젝트는 카카오톡 챗봇, OCR, Google Sheets API를 결합하여 출석 인증, 벌금 정산, 스토리지 관리를 자동화하는 **Serverless 정산 시스템**입니다.

---

## 1. 🏗 System Architecture
* **Backend**: Python 3.11 / FastAPI (Asynchronous)
* **Database**: Google Sheets API (Source of Truth / NoSQL Interface)
* **Storage**: Google Drive API (Image Hosting & Lifecycle)
* **OCR**: Google Cloud Vision API (Parsing Gurumi UI)
* **Infrastructure**: Railway (PaaS), 카카오 오픈빌더 스킬 연동

---

## 2. 📑 핵심 운영 로직 정리

### 2.1 🕒 시간 검증 및 마감 룰 (Time Logic)
* **공부 인정 시간**: 당일 05:00 ~ 익일 01:00 (새벽 1시까지 목표 달성 완료 필수)
* **예외 인정**: 01:00을 넘겨서 공부가 끝났더라도, 타이머상 목표 시간을 달성했다면 인정 (예: 01:20 종료, 타이머 2h 10m / 목표 2h -> 인정).
* **제출 마감**: 익일 정오(1:00) (이후 제출 시 이유 불문 결석 처리 및 벌금 -2,000원).
* **종료 시간 판독**:
  * 사용자가 공부를 마치자마자 사진을 찍어서 보내는 것을 원칙으로 함.
  * 카카오톡 전송 시각을 시스템상 종료 시각으로 간주. (사진이 챗봇 서버에 들어온 Webhook Time이 01:00 이전이면 무조건 세이프)
  * 만약 01:00 이후(예: 오전 8시)에 사진을 보냈다면, 시스템은 이를 **"01:00 이후에 공부를 마친 것"**으로 간주하여 사진 속 공부 시간을 엄격히 대조 및 벌금 부과 심사 (정시 전송 유도).

### 2.2 🔍 판독 및 부정 방지 로직 (OCR & Anti-Cheat)
* **권장 인증 사진 양식**: **[구루미 출첵 화면 '닉네임/공부시간/누적시간']**
  * 닉네임(목표시간)이 한 화면에 나옴 (OCR 매칭 및 수동 확인 용이).
  * 당일 공부 시간이 크게 표시됨 (OCR 판독률 최고).
  * 누적 시간(Total)이 표시됨 (부정 인증 방지).
* **목표 시간 설정**:
  * 유저 닉네임 파싱: 이름(3h) -> 3시간 / 괄호 없을 시 기본 2시간.
  * 반휴 사용 시: 원래 목표와 상관없이 당일 목표 1시간으로 고정.
* **거짓 인증 체크**:
  * 사진 속 **[누적 시간]**을 추출하여 시트 내 최종 누적시간과 비교.
  * 현재 누적 < 이전 누적일 경우 벌금 -5,000원 자동 부과 및 관리자 알림.
* **닉네임 파싱 로직**:
  * 카카오톡 챗봇은 사진을 보낸 사람의 고유 ID(UserKey)를 함께 보냄. 최초 1회만 닉네임과 UserKey를 연결.
  * 시스템은 이후 UserKey로 100% 관리하며, 사진 속 닉네임은 방장 대조용으로만 활용.

### 2.3 💰 벌금 및 제로섬 정산 로직 (Zero-Sum Engine)
* **벌금 산정**:
  * **결석**: -2,000원 (미인증 또는 익일 정오 초과 시점)
  * **시간 미달 (1시간 이상 인증)**: -500원
  * **시간 미달 (1시간 미만 인증)**: -1,000원
  * **허위 인증 (날짜 등)**: -1,000원
  * **거짓 인증 (데이터 조작 등)**: -5,000원
* **주간 결산 (매주 토요일 12:00)**:
  * 배분: (한 주간 쌓인 총 벌금) / (벌금이 0원인 성실 멤버 수) 만큼 예치금 플러스.
  * 출금 알림: 상금이 포함된 예치금이 13,000원 이상(+3,000원 수익)인 경우, 방장에게 입금 리스트 전달 및 해당 인원 예치금 10,000원으로 수동 리셋.
  * 고갈 알림: 예치금 0원 이하 인원에게 자동 경고 메시지 발송.

## 3. 🤖 챗봇 사용자 경험 (UX Flow)
* **사진 전송**: 1:1 채팅방에 사진 업로드.
* **즉시 응답**: 카카오 챗봇 5초 타임아웃 방지를 위해 "인증을 분석 중입니다. 잠시만 기다려주세요!" 메시지 선발송.
* **결과 통보**: 분석 완료 후 **[당일 시간 / 누적 시간 / 판정 결과 / 현재 예치금]**을 담은 카드 메시지 발송.
* **메뉴 제공**: 하단 퀵 메뉴로 반휴/주휴/특휴 신청 및 내 현황 조회 기능 제공. 특휴 신청 받을 경우 승인 처리 여부 포함.

## 4. 🗂 Database & Storage Schema (Google API)
* **Source of Truth**: Google Sheets (DB 역할)
* **Storage**: Google Drive (인증 사진 저장 후 URL만 시트에 기록. 매주 일요일, 14일 지난 사진 자동 삭제로 용량 관리 Zero)

### Sheet 1: `Member_Master` (관리자용)
활동 중인 인원만 필터링하여 체크. 정산 후 방장님이 상금 입금 후 10,000원으로 수동 복구.

| 닉네임 | UserKey | 상태 | 목표시간 | 최종누적 | 주간휴무 | 남은월휴 | 예치금 | 비고 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| momofa(2h) | UK123 | 활동 | 120 | 15,600 | 1.0 | 1 | 10,000 | - |
| 버디(3h) | UK456 | 휴가 | 180 | 10,000 | 1.0 | 1 | 13,500 | 상금대상 |

*(💡 휴무 자동 차감: 챗봇에서 반휴/주휴/월휴 사용 시 봇이 이 시트의 잔여량을 체크 후 자동으로 차감(주휴 1.0, 반휴 0.5, 월휴 1)합니다.)*

### Sheet 2: `Admin_Config` (스위치 및 공지)
공휴일 스위치 및 주간 공지사항 텍스트 관리.

| 날짜 | 이벤트 타입 | 목표시간 조정 | 주간 공지사항 (추가 멘트) |
| :--- | :--- | :--- | :--- |
| 2026-05-05 | 자율참여 | 0 | 어린이날 즐겁게 보내세요! |
| 2026-05-15 | 자정이후점검 | 60 | 점검으로 인해 오늘 목표 1시간입니다. |

### Sheet 3: `Daily_Log` (트랜잭션)
모든 인증 트랜잭션 기록.

| 날짜 | 닉네임 | 유형 | 판정 | 승인여부(특휴시) | 당일시간 | 사진누적 | 벌금액 | 이미지ID |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 2026-04-15 | momofa | 일반 | PASS | - | 135 | 15,600 | 0 | drive_id_1 |
| 2026-04-16 | 버디 | 특휴 | 대기 | N | - | - | 0 | url_link |

## 5. 🛠 운영 관리 및 예외 상황 처리 (Admin Control & Switch)
* **휴무 자동 차감 및 방어 (자동화)**: 사용자가 챗봇에서 반휴/주휴/월휴 버튼을 누르면 봇이 자동으로 잔여 횟수를 검사하고 차감합니다. (주휴 사용자는 주간휴무 잔여량이 0이 되어, 반휴를 추가로 누르더라도 시스템이 알아서 거절합니다)
* **특휴 승인 (관리자 수동)**: 증빙이 필요한 특휴(병가 등) 접수 시 `Daily_Log` 시트에 `대기`, `승인여부: N` 으로 기록됩니다. 방장님은 시트 확인 후 `승인여부`를 **Y**로, 판정을 **PASS**로만 변경해 주시면 됩니다. (`Member_Master`의 상태를 일부러 건드릴 필요가 없습니다!)
* **전체 자율 참여**: 공휴일이나 서버 점검 시 `Admin_Config` 기입 -> 해당 날짜는 벌금 로직 가동 중단 및 안내 멘트 자동 변경.
* **자정 이후 점검**: 목표 시간 일괄 1시간으로 자동 하향 조정 (반휴 적용 불가, 미달 시 -500원).
* **개별 오류**: 관리자가 `Daily_Log` 시트에서 벌금액을 직접 수정(0원 등)하면 정산 엔진이 수정된 값을 자동 반영.

## 6. 📬 획기적인 주간 결산 메시지 (자동 생성 템플릿)
매주 토요일 정오에 자동 발송. 방장님이 일일이 치지 않고 시트(`Admin_Config`)의 '주간 공지사항' 칸만 수정하면 자동 반영되도록 설계.
```text
[Study-Sync 주간 결산] 📅 {시작일} ~ {종료일}

이번 주도 고생 많으셨습니다! 
정산 결과를 발표합니다.

💰 벌금 합계: {Total_Penalty}원
💰 1/n 배분액: +{N_Reward}원 (성실멤버 {Count}명)

🏆 명예의 전당 (최장 시간)
1위: {User1} ({Time1}시간)
2위: {User2} ({Time2}시간)

💸 상금 입금 (3,000원↑) :
- {UserA}님: {AmountA}원
- {UserB}님: {AmountB}원
* 방장님이 입금 후 예치금을 10,000원으로 복구할 예정입니다.

⚠️ 예치금 부족 (입금 필요) :
- {UserC}님 (현재 {Balance}원)

📢 관리자 공지사항:
-------------------------
{Admin_Notice_Field}
-------------------------
```
